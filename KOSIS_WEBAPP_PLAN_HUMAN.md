# KOSIS 웹앱 기획안 (사람용)

## 1. 프로젝트 개요
- 프로젝트명: KOSIS Insight Hub (가칭)
- 목적: 기존 `runner.py + jobs/*.json + output/*.xlsx` 배치 구조를 웹앱 형태로 전환해, 비개발자도 데이터 수집/가공/다운로드를 쉽게 수행하도록 한다.
- 현재 기준일: 2026-05-11

## 2. 문제 정의
- 배치 파일 실행(`run_*.bat`) 중심이라 진입 장벽이 높다.
- 실행 상태/실패 원인을 실시간으로 확인하기 어렵다.
- 결과 파일은 생성되지만, 웹에서 탐색/다운로드/이력 관리가 어렵다.

## 3. 목표
- 목표 1: 클릭 기반 실행 UI 제공 (카테고리/잡 선택, 전체 실행, 선택 실행)
- 목표 2: 실행 로그 및 진행률 실시간 표시
- 목표 3: 산출물(`output/*.xlsx`) 탐색 및 다운로드
- 목표 4: 운영 안정성(재시도, 에러 표시, 실행 이력)을 기본 제공

## 4. 비목표 (초기)
- 멀티테넌트 권한/결재 워크플로우
- 외부 BI 대시보드 완전 대체
- 대규모 분산 처리 아키텍처

## 5. 사용자
- 운영자: 정기/수시 데이터 추출 실행
- 분석가: 최신 결과 파일 확인 및 다운로드
- 관리자: API 키/실행 정책 관리

## 6. 핵심 기능 요구사항
- 기능 A: 잡 탐색
- 기능 B: 실행 제어 (전체/카테고리/개별 잡)
- 기능 C: 실행 상태 (대기/실행/성공/실패)
- 기능 D: 로그 스트림 (시간, 잡명, 레벨, 메시지)
- 기능 E: 결과 파일 목록/검색/다운로드
- 기능 F: 환경 설정 (API 키, 타임아웃, 재시도)

## 7. 정보 구조 (IA)
- 메뉴 1: Dashboard
- 메뉴 2: Run Jobs
- 메뉴 3: Results
- 메뉴 4: Logs
- 메뉴 5: Settings

## 8. 화면 기획 요약
- Dashboard: 오늘 실행 건수, 성공률, 최근 실패 건수
- Run Jobs: 잡 체크박스, 실행 버튼, 진행률 바
- Results: 파일 목록, 생성일, 크기, 다운로드
- Logs: 실시간 로그 + 필터(레벨/잡)
- Settings: 실행 옵션 저장, 키 상태 확인

## 9. 기술 권장안 (웹앱형)
- UI: Next.js
- API/Worker: FastAPI
- 실행엔진: 기존 `runner.py` 재사용(서비스 계층 래핑)
- 저장소:
  - 메타: SQLite 또는 PostgreSQL
  - 산출물: 로컬 `output/` (초기), 추후 Object Storage 확장

## 10. 아키텍처 개요
- Frontend(Next.js) -> Backend(FastAPI) -> Runner Service(`runner.py`) -> KOSIS API
- Backend는 실행 요청, 상태 조회, 로그 스트림, 파일 다운로드 API 제공

## 11. 단계별 구현 로드맵
- Phase 1 (1~2주): 현재 Streamlit UI 고도화, 실행 연결, 기본 로그/결과 파일
- Phase 2 (2~4주): FastAPI 분리, 실행 큐/이력 DB, 오류 처리 표준화
- Phase 3 (4~6주): Next.js 전환, 권한/운영 편의 기능 추가

## 12. 성공 지표 (KPI)
- 작업 실행 성공률 95% 이상
- 실패 원인 식별 시간 10분 이내
- 비개발자 단독 실행 비율 80% 이상
