# Job Scope Classification Guide

`jobs/<part>/` 아래 JSON 파일을 어떤 범위 폴더에 둘지 빠르게 판단하기 위한 기준 문서입니다.

## 기본 원칙
- 1차 분류는 `part`
- 2차 분류는 `scope`
- 실제 판정 기준은 폴더명보다 JSON 메타가 우선

권장 메타:

```json
{
  "scope_type": "common",
  "scope_all_regions": true,
  "scope_regions": []
}
```

```json
{
  "scope_type": "incheon",
  "scope_all_regions": false,
  "scope_regions": ["인천"]
}
```

```json
{
  "scope_type": "multi_region",
  "scope_all_regions": false,
  "scope_regions": ["서울", "인천"]
}
```

## 폴더 규칙

```text
jobs/
  <part>/
    common/
    incheon/
    multi_region/
```

```text
output/
  <part>/
    common/
    incheon/
    multi_region/
```

## Scope 정의

### `common`
- 전국 비교
- 광역시도 비교
- 시도/권역 공통 구조
- 특정 지역 하나를 깊게 보는 작업이 아님
- 나중에 다른 지역에서도 그대로 재사용 가능

대표 예:
- 전국 산업별 사업체 수 비교
- 시도별 고용률 비교
- 광역시도 관광객 추이 비교

판정 질문:
1. 이 결과를 서울, 부산, 인천 어디에 두어도 어색한가?
2. 특정 지역 하나보다 전국/광역 비교가 핵심인가?
3. 지역을 바꿔도 동일한 템플릿으로 재사용 가능한가?

세 질문 중 2개 이상 `예`면 `common` 추천.

### `incheon`
- 인천 단독 분석
- 인천 군구/행정구역 세부 비교
- 인천 특성별/계층별/세부 항목 분석
- UI에서 인천 선택 시 보여주는 것이 가장 자연스러운 작업

대표 예:
- 인천 군구별 인구 변화
- 인천 산업단지 현황
- 인천 취업자 특성별 분석
- 인천 관광지 방문 패턴

판정 질문:
1. 제목에 인천이 직접 들어가는가?
2. 인천 하위 행정구역이나 인천 내부 비교가 중심인가?
3. 다른 지역으로 그대로 옮기면 의미가 크게 달라지는가?

세 질문 중 2개 이상 `예`면 `incheon` 추천.

### `multi_region`
- 특정 두세 지역을 묶어서 비교
- 전국 공통이라 보기엔 범위가 좁고, 인천 전용도 아님
- 수도권, 서울+인천, 영남권처럼 명시적 지역 묶음

대표 예:
- 서울+인천 청년 순이동
- 수도권 산업단지 비교
- 부산+울산 물류/고용 비교

판정 질문:
1. 대상 지역이 2개 이상인가?
2. 전국 전체 비교는 아니지만 특정 묶음 비교인가?
3. 특정 권역/생활권 단위가 분석 핵심인가?

세 질문 중 2개 이상 `예`면 `multi_region` 추천.

## 파트별 추천 기준

## `economy_industry`

### `common`으로 두기 좋은 유형
- 전국/광역시도 산업구조 비교
- 시도별 사업체 수, 종사자 수, 제조업 비중 비교
- 전국 공통 산업 KPI

### `incheon`으로 두기 좋은 유형
- 인천 산업단지
- 인천 항만/공항 연계 산업
- 인천 군구별 사업체/종사자
- 인천 지역내총생산 또는 인천 특화산업 분석

### `multi_region`으로 두기 좋은 유형
- 인천 vs 서울 vs 경기
- 인천 vs 부산 항만도시 비교
- 수도권 산업 분포 비교

## `employment_labor`

### `common`으로 두기 좋은 유형
- 시도별 고용률/실업률 비교
- 전국 공통 노동시장 지표
- 광역시도 청년고용 비교

### `incheon`으로 두기 좋은 유형
- 인천 취업자 특성별
- 인천 청년/여성/고령층 노동 분석
- 인천 군구별 고용지표

### `multi_region`으로 두기 좋은 유형
- 서울+인천 출퇴근권 고용 비교
- 수도권 청년고용 비교
- 인천 vs 부산 vs 울산 제조업 고용 비교

## `tourism_culture`

### `common`으로 두기 좋은 유형
- 시도별 관광객 수 비교
- 광역시도 문화시설 비교
- 전국 공통 관광/문화 KPI

### `incheon`으로 두기 좋은 유형
- 인천 관광지 방문객
- 인천 문화시설/행사/체육시설
- 인천 구별 문화 접근성

### `multi_region`으로 두기 좋은 유형
- 인천공항 연계 수도권 관광 비교
- 서울+인천 관광 이동 비교
- 서해권 관광권역 비교

## 애매한 경우 우선순위
1. 전국/광역 구조면 `common`
2. 인천 내부 분석이면 `incheon`
3. 특정 복수 지역 비교면 `multi_region`

즉, 애매할 때는 아래 순서로 판단합니다.

```text
전국/광역 공통인가?
  yes -> common
  no  -> 인천 단독인가?
           yes -> incheon
           no  -> 복수 지역 비교인가?
                    yes -> multi_region
                    no  -> incheon 우선 검토
```

## 새 JSON 추가 체크리스트
- 파일 위치가 `jobs/<part>/<scope>/` 규칙을 따르는가
- `scope_type`이 폴더와 일치하는가
- `scope_regions`가 실제 대상 지역과 일치하는가
- `output_subdir`가 `output/<part>/<scope>` 구조와 맞는가
- 제목/설명만 보고도 범위가 이해되는가

## 운영 팁
- 처음엔 너무 세분화하지 말고 `common / incheon / multi_region` 3개만 유지
- `seoul`, `busan` 같은 단독 지역 폴더는 실제 JSON이 충분히 쌓일 때 추가
- 파일명에 지역명이 들어가더라도 전국 비교 작업이면 `common` 유지
- 폴더보다 메타가 최종 진실값이라는 원칙을 계속 유지
