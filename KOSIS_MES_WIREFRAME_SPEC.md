# KOSIS MES 와이어프레임 스펙

## 1. 전체 레이아웃
```text
+----------------------------------------------------------------------------------+
| Header: Logo | Env | Clock | Alarm | User                                       |
+----------------------+--------------------------------------+--------------------+
| Sidebar              | Main Center Panel                    | Right Event Panel  |
| - Dashboard          | - KPI Row                            | - Alerts          |
| - Work Orders        | - Work Order Detail                  | - Recent Errors   |
| - Monitor            | - Step Progress                       | - Retry Queue     |
| - Artifacts          | - Action Buttons                      |                   |
| - Logs               |                                      |                   |
| - Settings           |                                      |                   |
+----------------------+--------------------------------------+--------------------+
| Bottom Console: Live Log Stream + Filter + Search + Export                        |
+----------------------------------------------------------------------------------+
```

## 2. 화면별 컴포넌트 상세

### Dashboard
- KPI 카드 4개: 금일 실행, 성공률, 실패 건수, 평균 처리시간
- 카테고리 상태 매트릭스(행=카테고리, 열=상태)
- 최근 실패 10건 테이블

### Work Orders
- 좌측: 지시 목록 테이블
  - 컬럼: ID, 대상, 상태, 우선순위, 시작시각, 소요시간
- 중앙: 선택 지시 상세
  - 실행 대상 잡 리스트
  - 단계 진행률 바
  - 실행/중지/재시도 버튼
- 우측: 알람/이벤트
  - 실패 이벤트 카드
  - 즉시 재처리 버튼

### Monitor
- 현재 RUNNING 건 카드
- 단계별 간트(차기)
- ETA 및 병목 단계 표시

### Artifacts
- 파일 테이블
  - 파일명, 실행ID, 생성시각, 크기, 다운로드
- 필터: 카테고리, 날짜, 상태

### Logs
- 실시간 스트림
- 레벨 토글(INFO/WARN/ERROR)
- run_id 검색
- 로그 내보내기(.txt/.json)

## 3. 디자인 토큰 (MES 느낌)
- 배경: `#0B1220`
- 패널: `#111A2E`
- 경계: `#22314E`
- 기본 텍스트: `#E6EDF7`
- 보조 텍스트: `#9DB0CC`
- 성공: `#22C55E`
- 경고: `#F59E0B`
- 실패: `#EF4444`
- 강조: `#38BDF8`

## 4. 인터랙션 규칙
- RUNNING 상태에서는 동일 WorkOrder 중복 실행 버튼 비활성화
- FAILED 항목은 빨간 상태칩 + "재시도" 버튼 노출
- 로그는 기본 자동스크롤 ON, 사용자가 스크롤하면 자동 OFF
- 알람 클릭 시 해당 WorkOrder 상세로 포커스 이동

## 5. 반응형 규칙
- Desktop(>=1280): 3패널 + 하단 콘솔
- Tablet(768~1279): 우측 이벤트 패널 접기 가능
- Mobile(<768): 탭 전환 방식으로 패널 분리

## 6. 개발 우선순위
1. 레이아웃 틀(헤더/사이드/3패널/콘솔)
2. 상태칩 및 KPI 카드 공통 컴포넌트
3. WorkOrder 목록 + 상세 동기화
4. 실시간 로그 콘솔
5. 알람 패널 및 재시도 액션
